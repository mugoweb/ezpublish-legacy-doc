[ArchiveSettings]

SettingSummary
Handler DEPRECATED since eZ Publish 4.2.

[BinaryFileSettings]

SettingSummary
ExtensionRepositories DEPRECATED since eZ Publish 4.1
Handler This setting sets which binary file handler to use.
Repositories DEPRECATED since eZ Publish 4.1

[ClusterEventsSettings]

SettingSummary
ClusterEvents Enables/Disables dedicated cluster events listening.
Listener Defines which class to use as cluster events listener.

[ClusteringSettings]

SettingSummary
DBChunkSize This determines the byte size of the blocks into which files are split when they are fetched and inserted from the database, when clustering with the eZDB File Handler.
ExtensionDirectories DEPRECATED since eZ Publish 4.1.
FileHandler This setting sets the cluster file handler.
NonExistentStaleCacheHandling This setting defines what happens when a requested cache file is already being generated and no expired cache file exists.

[eZDFSClusteringSettings]

SettingSummary
DBBackend Sets the database back-end which should be used by the eZ DB File Handler.
DBConnectRetries This sets the permitted amount of connection retries when trying to connect to the database, when clustering with the eZDB File Handler.
DBExecuteRetries This sets the permitted amount of executer retries.
DBHost Sets which database host to use when clustering with the eZDB File Handler.
DBName This sets which database to use when clustering with the eZ DB File Handler.
DBPassword This sets which password to use for your database when clustering with the eZDB File Handler.
DBPort This setting sets which port to use when clustering with the eZDB File Handle
DBSocket Enables the use of sockets by eZ DFS FIle Handler.
DBUser This sets which username to use for your database when clustering with the eZDB File Handler.
MaxCopyRetries This setting defines the maximum number of times a DFS file is tried to be copied.
MountPointPath This setting defines the path to the NFS Mount Point, which should be used when clustering with an eZ DFS File Handler.

[FileSettings]

SettingSummary
Handlers Use this setting to define which handler to use.

[PassThroughSettings]

SettingSummary
ContentDisposition Controls the way binary files should be served over HTTP, either as downloadable as an attachment, or displayed inline in the browser.
