System requirements and issues

  •  eZ Publish version 5.0 
     
    •  needs to be set up with eZ jscore extension 5.0.
    •  the Comments module does not work on a plain site (The 'posting comment' feature will not work because of the plain site template). So, Website Interface or eZ Flow extension are required.
  •  The administrator needs to set up a cronjob to search for new subscriptions. How to set up a cronjob is described in the chapter Cronjobs of the eZ Publish documentation.

Important notes:

There are two known issues when using this module on an eZ Publish version before 4.3:

  •  The first occurs when a user is granted permission to edit or delete his own comment. They will be able to see all the edit and delete buttons on the comment list, although he can not edit or delete all the comments.
  •  A second smaller issue is that, in administrator GUI, posting comments doesn't work in the content preview tab, but editing and deleting does.

Performance note:

The Comments module uses the same cache mechanism for the comment page as in the eZ Publish content. If a page containing comments is visited once, it will be cached and in the next visit it will fetch data from the cache storage instead of the database.
